Quiero que MIGRES este proyecto (actualmente maquetado en HTML + CSS + JS puro) a una aplicación en **Angular 19** usando **componentes standalone**.

### Contexto del proyecto actual

- Ya existe un prototipo de kiosko de control de acceso hecho en:
  - `index.html` (estructura de pantallas)
  - CSS (estilos de la UI tipo kiosko)
  - JavaScript (lógica básica de cambio de pantallas y modales)
- El diseño ya está definido: pantallas para lectura de QR, captura de identificación y mensaje de acceso autorizado.

Tu tarea es:

---

## 1. Crear un proyecto Angular 19 standalone

1. Genera un proyecto Angular dentro de este mismo Repl:
   - Estructura estándar de Angular 19 con TypeScript.
   - Usa **componentes standalone** (sin NgModules).
2. Asegúrate de:
   - Tener un `main.ts` que haga `bootstrapApplication(AppComponent)`.
   - `AppComponent` sea `standalone: true`.
   - No usar módulos clásicos (`NgModule`), todo debe ser standalone.

Puedes crear una carpeta `kiosk-app` o similar para el proyecto Angular, y dejar claro cuál es el punto de entrada.

---

## 2. Migrar la maquetación existente a componentes Angular

1. Mantén **el mismo look & feel** que ya tengo en el HTML/CSS original.
2. Crea la siguiente estructura mínima de componentes standalone:

   - `AppComponent`:
     - Componente raíz, contendrá el layout general y el logo.
     - Incluirá al componente `KioskComponent`.

   - `KioskComponent`:
     - Contendrá la lógica de la máquina de estados del kiosko.
     - Estados:
       - `pantalla1` → bienvenida + lectura de QR.
       - `pantalla2` → captura de identificación + modal de datos INE.
       - `pantalla3` → mensaje de acceso autorizado.
     - Usa `ngSwitch` o una lógica similar para mostrar cada pantalla.

   - Opcional (si ayuda a organizar mejor):
     - `PantallaQrComponent`
     - `PantallaIdentificacionComponent`
     - `PantallaAccesoAutorizadoComponent`
     - `ModalIdentificacionComponent`
     - Todos estos deben ser `standalone: true` y se importan en el `KioskComponent`.

3. Migra el CSS:
   - Combina el CSS actual en hojas de estilo de Angular (puede ser `styles.css` global o estilos por componente).
   - Respeta la idea original: pantalla kiosko, fondo oscuro, tipografía grande, tarjetas, etc.
   - Asegúrate de que NO haya scroll en el body; todo debe caber en una pantalla de 7" horizontal (tipo 1024x600).

## 3. Implementar la cámara con stream continuo y lector de QR automático

Quiero que el kiosko funcione de forma **autónoma**: la cámara está encendida y el QR se lee automáticamente, SIN necesidad de que el usuario presione un botón.

1. En el `KioskComponent` (o `PantallaQrComponent`) agrega:
   - Un `<video #videoElement>` para mostrar el stream de la cámara.
   - Un `<canvas #canvasElement>` oculto (solo para procesar frames).

2. En TypeScript:
   - Usa `@ViewChild('videoElement')` y `@ViewChild('canvasElement')` para acceder a los elementos.
   - En `ngOnInit` o `ngAfterViewInit`:
     - Solicita acceso a la cámara con `navigator.mediaDevices.getUserMedia({ video: { facingMode: 'environment' } })`.
     - Asigna el stream al `videoElement.srcObject` y llama a `play()`.

3. Lectura de QR:
   - Usa una librería de QR para navegador, por ejemplo `jsqr` o `@zxing/browser`.
   - INSTALA la librería vía npm dentro del proyecto Angular y úsala desde el componente.
   - Implementa un ciclo de lectura tipo:
     - Cada 250 ms:
       - Dibujar el frame actual del video en el canvas.
       - Intentar decodificar el QR.
     - Si se detecta un QR:
       - Detener el proceso de decodificación (pero NO cerrar la cámara).
       - Guardar el valor del QR.
       - Avanzar al flujo de validación (simular llamada a API).

4. Reglas importantes:
   - La cámara se mantiene **SIEMPRE encendida** mientras el kiosko esté activo.
   - Solo el proceso de decodificación se pausa cuando ya se detectó un QR.
   - Cuando el flujo termina y se regresa a la pantalla inicial, se reactiva el decodificador para leer el siguiente QR.

## 4. Mantener y mejorar la máquina de estados del kiosko

Implementa en Angular un servicio o una lógica directa en `KioskComponent` para manejar los estados:

Estados:
- `qrIdle` → Esperando QR (Pantalla 1, sin spinner).
- `qrValidando` → Spinner “Validando en sistema…”.
- `qrValidado` → Mensaje “Qué tal, Erick… tu código QR es válido…”.
- `esperandoIdentificacion` → Pantalla 2 esperando botón “Capturar identificación”.
- `leyendoIdentificacion` → Spinner “Leyendo y validando identificación…”.
- `identificacionMostrada` → Modal con datos de INE (nombre, fecha de nacimiento, dirección, sexo, CURP).
- `accesoAutorizado` → Pantalla 3 con “Acceso autorizado” y mensajes de recomendaciones.

Comportamiento:

1. `qrIdle`:
   - Cámara encendida.
   - Escucha QR automáticamente.
   - Al detectar un QR → `qrValidando`.

2. `qrValidando`:
   - Muestra spinner y texto.
   - Tras 3 segundos (simulación de API) → `qrValidado`.

3. `qrValidado`:
   - Muestra saludo a Erick y rango de fechas del QR.
   - Tras otros 3 segundos → `esperandoIdentificacion`.

4. `esperandoIdentificacion`:
   - Pantalla que indica “Coloca tu identificación…”.
   - Botón grande `Capturar identificación`.
   - Al hacer clic → `leyendoIdentificacion`.

5. `leyendoIdentificacion`:
   - Spinner de lectura de identificación.
   - Tras 3 segundos → `identificacionMostrada`.

6. `identificacionMostrada`:
   - Mostrar modal centrada con los datos simulados de la INE.
   - Botón `Confirmar y continuar` → `accesoAutorizado`.

7. `accesoAutorizado`:
   - Mostrar la pantalla de “Acceso autorizado” (como en el diseño actual).
   - 3 segundos después → regresar a `qrIdle`:
     - Cerrar modal si sigue abierta.
     - Limpiar textos temporales.
     - Reactivar el decodificador de QR (la cámara ya está encendida).

## 5. Detalles de UX y responsividad para kiosko

- Todo debe visualizarse correctamente sin scroll en una pantalla de 7" horizontal.
- Si el contenido de la modal es largo, SOLO la modal debe tener `overflow-y: auto;`, el body principal no debe desplazarse.
- Botones grandes y centrados, fáciles de tocar.
- Mensajes claros en español, como en la maqueta original:
  - “Coloca tu código QR en el lector”.
  - “Validando en sistema…”.
  - “Datos de la identificación validados con éxito. Ya puedes pasar.”
- Utiliza animaciones suaves (por ejemplo, `opacity` y `transform`) para mostrar/ocultar las pantallas y la modal.
